Implementation Guideline — Encryption Operations
Version 2

This guideline operationalizes the Acceptable Use Policy, Section 2.

Implementation Note 1
Engineering teams must use the central key management service (KMS) for all
data-at-rest encryption keys. Local key storage is prohibited except for
ephemeral session keys.

Implementation Note 2
Database-level encryption (TDE) satisfies the at-rest requirement for
structured data. Object storage must use server-side encryption with
customer-managed keys.

Implementation Note 3
Key rotation must occur at least every 365 days. Compromise events require
immediate rotation and incident review.

Implementation Note 4
All TLS endpoints must use forward-secret cipher suites. Endpoints that
cannot enforce TLS 1.3 must be documented as exceptions with a remediation
date.
